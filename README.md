# SurgiScore Clinical Workspace v11

واجهة مبسطة لإدارة مرضى الجراحة، مبنية فوق وظائف وقاعدة بيانات v10 مع الحفاظ على التوافق.

## لماذا v11؟

الإصدار السابق كان غنياً بالوظائف لكنه يعرض وحدات كثيرة في التنقل اليومي. v11 يعيد تنظيم النظام حول عمل المستخدم والمريض بدلاً من قائمة Modules طويلة.

## التنقل الرئيسي

خمس مساحات فقط:

1. **اليوم Today** — المهام والنتائج والمواعيد والعمليات التي تحتاج إجراء الآن.
2. **المرضى Patients** — ملف موحد بخمس تبويبات: Overview, Plan, Timeline, Results, Documents.
3. **العمليات Surgery** — قائمة اليوم والقادمة والتقويم، ثم مسار واحد لكل عملية.
4. **الردهة Ward** — NEWS2، العلامات الحيوية المستحقة، Quick observations وQuick ward round.
5. **المزيد More** — العيادة، الوصفات، النتائج، المتابعة، السكورات، الجودة والحوكمة.

## أهم التحسينات

- Sidebar مختصر للآيباد والحاسبة.
- Bottom navigation للموبايل.
- Global search وQuick add ثابتان.
- Patient workspace موحد.
- Surgery workflow يعرض المرحلة الحالية والخطوة التالية والنواقص.
- Ward board كثيف ومناسب للجولات.
- تسجيل NEWS2 وجولة سريعة من الواجهة الرئيسية.
- إخفاء الوحدات الأقل استخداماً داخل More بدون حذفها.
- دعم الواجهات العربية مع بقاء المصطلحات الطبية الإنجليزية.
- متوافق مع قاعدة بيانات v9/v10؛ لا يتطلب حذف البيانات أو إنشاء جداول جديدة.

## التشغيل على Streamlit Community Cloud

- Main file: `app.py`
- Python: 3.11 أو 3.12
- ارفع جميع ملفات Runtime إلى جذر Repository.
- لا ترفع ملفات `.db`, `.sqlite`, `.sqlite3`.

```toml
APP_MODE = "demo"
HOSPITAL_NAME = "Surgical Department Pilot"

```

## تنبيه

Streamlit Community Cloud مناسب للتجربة والتحقق من سير العمل فقط. الاستخدام السريري ببيانات حقيقية يحتاج استضافة مؤسسية، PostgreSQL مُداراً، OIDC/MFA، تخزيناً خاصاً للمرفقات، نسخاً احتياطية، مراجعة أمنية واعتماداً سريرياً.
